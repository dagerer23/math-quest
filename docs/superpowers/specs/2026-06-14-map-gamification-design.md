# 地图关卡游戏化改造设计

## 背景

当前地图页面（Home.tsx）使用浅色背景 + Z字形路径 + 圆形数字节点，视觉上更像列表而非游戏地图，缺乏沉浸感和探索欲。本次改造目标是在最小改动范围内，让地图具备手游级的视觉体验和游戏感。

## 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 视觉风格 | 卡通可爱风 | 用户偏好，与多邻国风格一致 |
| 参考对象 | Duolingo 地图 | 用户明确参考 |
| 背景方案 | 无限渐变深色 | 关卡无限，背景需可延续；深色更有沉浸感 |
| 章节标识 | 无拱门/名称，纯渐变过渡 | 用户要求，渐变本身暗示章节切换 |
| 宝箱/每日 | 不做 | 用户要求精简 |
| 旗帜装饰 | 不做 | 用户要求精简 |
| 实现范围 | 最小可用版 | 聚焦核心视觉提升，快速上线 |

## 改造内容

### 1. 无限渐变背景

**现状：** 浅色纯色背景（`#F0F7F5` 等），按年级切换 3 套主题色。

**改造：** 一条连续的 CSS 渐变色带，深色系，随关卡推进自然过渡色域。

```
深绿(森林) → 暖沙(沙漠) → 冰蓝(冰雪) → …可无限追加
```

- 每个"色域"对应一段渐变色标，章节间用 5-8% 的过渡区自然衔接
- 新增章节只需在 `linear-gradient` 末尾追加色标，无需图片资源
- 渐变色标存储在 THEMES 配置中，按 grade 扩展

**场景装饰（随色域自动出现）：**
- 森林段：萤火虫粒子（CSS animation）
- 沙漠段：沙尘粒子
- 冰雪段：雪花粒子
- 场景剪影（SVG）：树木/仙人掌/冰山，用低透明度 SVG 绘制

### 2. 路径系统升级

**现状：** 细虚线/实线 SVG path，1px 粗细。

**改造：**
- 已完成路径：双层绘制 — 底层宽 14px 半透明光晕 + 上层 4px 实色 + SVG glow 滤镜
- 当前路径：渐变色过渡（绿→蓝）
- 未解锁路径：3px 虚线，越远越淡（opacity 递减）
- 流动光点：`<animateMotion>` 沿已完成路径移动，3s 循环

### 3. 关卡节点差异化

**现状：** 所有节点都是圆形 + 数字编号，仅靠边框颜色/粗细区分状态。

**改造为 4 种节点类型：**

| 类型 | 形状 | 尺寸 | 配色 | 特效 |
|------|------|------|------|------|
| 已完成 | 圆角方形 | 54px | 绿色渐变 `#34d399→#10b981` | 多层阴影+内阴影，对勾图标 |
| 当前 | 圆角方形 | 62px | 靛蓝渐变 `#818cf8→#6366f1` | 浮动动画+双层脉动光环+加号图标 |
| Boss | 圆角方形 | 70px | 橙色渐变 `#fb923c→#ea580c` | BOSS标签+光晕脉动+星形图标 |
| 锁定 | 圆角方形 | 46px | 毛玻璃 `rgba(255,255,255,0.06)` | backdrop-filter:blur + 锁图标 |

**图标统一使用 SVG 线条风格**（与 lucide-react 风格一致），不使用 emoji。

**星星行保留：** 3 星制，金色 `#E5A13B`，与现有一致。

**标签保留：** level.chapter 名称 + 状态描述（通关/进行中/题数），与现有数据结构兼容。

### 4. 角色陪伴

- 小精灵站在当前关卡旁边（偏右上方）
- 纯 CSS/SVG 绘制：圆形身体 + 眼睛 + 嘴巴
- 金色渐变 `#fbbf24→#f59e0b`，与现有星星色系一致
- 呼吸+弹跳动画（2s ease-in-out infinite）
- 可扩展：未来支持不同皮肤/宠物

### 5. 顶部 UI 适配

**StatusBar 适配深色：**
- 背景：`rgba(10,10,20,0.75)` + `backdrop-filter: blur(16px)`
- 文字/图标颜色改为白色系
- 数据结构不变：hearts, streak, xp, 签到按钮

**信息栏适配深色：**
- 年级徽章：毛玻璃背景 + 白色文字
- 每日目标按钮：毛玻璃背景
- 进度条：绿色填充保留 `#58CC02`

### 6. 底部导航适配

- 背景：`rgba(10,10,20,0.9)` + `backdrop-filter: blur(20px)`
- 激活色保留 `#58CC02`（duolingo-green）
- 结构不变：4 Tab（地图/错题/榜单/我的）

### 7. 心数弹窗

- 保留现有弹窗结构和逻辑
- 样式不变（白色卡片，在深色地图上对比度更好）

## 不改动的部分

- `getZigzagPositions` — Z字路径计算逻辑不变
- `enterLevel` — 关卡进入逻辑（心数检查、题目生成、导航）不变
- `getLevelStatus` — 关卡状态判断不变
- `useUserStore` — 用户数据结构不变
- `Level` / `UserState` 类型定义不变
- `PathConnector` 组件 — 升级视觉但接口不变
- `LevelNode` 组件 — 升级视觉但 props 不变
- 路由结构、Layout、BottomNav 组件结构不变

## THEMES 配置扩展

现有 THEMES 按 grade 定义浅色配色。改造后扩展为深色渐变配色：

```typescript
interface ThemeConfig {
  // 现有字段保留（用于弹窗等浅色元素）
  bg: string
  accent: string
  // ...
  
  // 新增：地图渐变色标
  gradientStops: { offset: number; color: string }[]
  // 新增：场景装饰类型
  particleType: 'firefly' | 'dust' | 'snowflake' | 'none'
  // 新增：场景剪影 SVG 标识
  silhouetteType: 'forest' | 'desert' | 'ice' | 'none'
}
```

## 实现优先级

1. **P0 — 背景渐变 + 节点差异化 + 路径升级**（核心视觉提升）
2. **P1 — 角色陪伴 + 场景装饰**（增加趣味性）
3. **P2 — StatusBar/BottomNav 深色适配**（整体一致性）

## 技术方案

- 背景渐变：CSS `linear-gradient`，色标从 THEMES 配置动态生成
- 节点差异化：LevelNode 组件内按 `isCompleted / isCurrent / isBoss / isUnlocked` 分支渲染
- 路径发光：SVG `<filter>` + 双层 path
- 流动光点：SVG `<animateMotion>`
- 角色陪伴：CSS animation + SVG 绘制
- 场景粒子：CSS `@keyframes` + 绝对定位 div
- 场景剪影：内联 SVG，低透明度
- 深色适配：StatusBar/BottomNav 背景改半透明+毛玻璃，文字改白色系
