# 知识点掌握度动态出题系统设计

## 概述

将当前固定题目关卡系统改造为基于知识点掌握度的动态出题系统，类似多邻国的自适应学习算法。地图按知识点分区域，每关动态生成题目，难度随用户表现自适应调整。

## 数据模型变更

### Question 新增字段
- `difficulty_score`: number (1-10)，精细难度分，后台可维护
- `knowledge_point`: string（已有），题目所属知识点

### Level 变更
- 移除 `questions: Question[]` 固定题目列表
- 新增 `knowledge_point_ids: string[]`，该关卡覆盖的知识点范围
- 新增 `target_mastery: number` (0-1)，通关所需最低掌握度，默认0.4
- 保留 `sortOrder`、`isBoss`、`grade`、`chapter` 等字段

### 用户数据
- 复用已有 `knowledgeProgress: Record<string, number>`，值范围 0.0~1.0
- 每次答题后更新对应知识点的掌握度

## 出题算法

### 输入
关卡的知识点范围 + 用户当前掌握度 + 近期答题历史

### 输出
8-10道最优题目

### 步骤
1. 计算每个知识点的优先级权重：
   - 未掌握(0-0.3): 权重3（重点出题）
   - 学习中(0.3-0.7): 权重2（适量出题）
   - 已掌握(0.7-1.0): 权重1（间隔复习）

2. 按权重分配每知识点出题数（总计8-10题）

3. 每知识点内选题：
   - 掌握度低 → 优先选 difficulty_score 低的题
   - 掌握度高 → 优先选 difficulty_score 高的题
   - 避免连续出同一题（记录近期答题历史）

4. Boss关特殊处理：混合多个知识点，难度偏高

## 掌握度更新算法

### 答对
- 简单题(difficulty 1-3): 掌握度 += 0.05
- 中等题(difficulty 4-6): 掌握度 += 0.08
- 困难题(difficulty 7-10): 掌握度 += 0.12

### 答错
- 掌握度 -= 0.10（最低不低于0）

### 连击奖励
- 连续答对3题同知识点：额外 +0.05

## 关卡解锁规则

- 第一关默认解锁
- 当前关卡内所有知识点的平均掌握度 >= target_mastery(默认0.4) 时解锁下一关
- Boss关需要前置所有知识点掌握度 >= 0.6

## 地图UI适配

- 每个关卡节点显示知识点名称（如"加法入门"、"进位加法"）
- 关卡节点上显示掌握度进度环（0-100%），替代原来的星星
- 通关条件：掌握度达标（而非答对X题）
- 已通关关卡可重做，继续提升掌握度
- Boss关用特殊样式标识

## 后台管理适配

- 题库管理：每题可编辑 difficulty_score 和 knowledge_point
- 关卡管理：每关可编辑 knowledge_point_ids 和 target_mastery
- 新增"难度分析"面板：显示每个知识点×难度组合的题目数量分布

## API变更

- `GET /api/content/level/:id` → 返回关卡信息（含知识点范围，不含固定题目）
- `POST /api/content/generate-questions` → 根据关卡+用户掌握度动态生成题目列表
- `POST /api/content/update-mastery` → 答题后更新掌握度（前端调用，后端计算）
