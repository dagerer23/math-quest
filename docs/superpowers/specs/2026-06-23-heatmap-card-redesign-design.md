# 学习日历卡片优化设计

**日期**: 2026-06-23
**状态**: 已确认待实施
**范围**: Taro 小程序端（`platform-taro`），Web 端同步

## 背景与问题

"我的"页面的学习日历卡片当前采用 30 天网格热力图（`HeatmapCalendar` 组件）。经评估存在以下问题：

1. **格子太密太小** —— 30 个格子挤在卡片内，数字和颜色难以辨识，缺少呼吸感
2. **信息层级平淡** —— 标题、统计数字、图例字号均为 10px，无视觉重点
3. **形式可优化** —— 密集网格在小程序竖屏体验欠佳
4. **缺少激励感** —— 仅有数据陈列，无连续打卡、本月坚持等激励反馈

## 目标

重新设计该卡片，使其同时承担**数据概览**与**激励坚持**双重角色，解决密度与层级问题，保留绿色系视觉语言。

## 不做（YAGNI）

- 不引入"题数目标"概念（不设置每周/每月题数目标值）
- 不使用 emoji（图标统一使用项目 `Icon` 组件）
- 不改动数据模型与后端接口（`DailyStat` 结构已足够）
- 不改动独立的学习统计页（`pages/stats`）

## 设计方案

### 整体布局

卡片自上而下分三区：

```
┌─────────────────────────────────────┐
│  [Icon] 学习日历          06.01-06.23│  ← 标题行
├─────────────────────────────────────┤
│  ╭─────╮  本月坚持得不错             │
│  │ 12  │  6月已过23天，你练习了12天   │  ← 进度环区
│  │活跃天│  继续保持，养成学习好习惯    │
│  ╰─────╯                            │
├─────────────────────────────────────┤
│  近 7 天            共34题 · 活跃5天 │
│  ▄ ▂  ▆ ▃ ▄ ▃ ▃                    │  ← 条形图区
│  一 二 三 四 五 六 日                │
└─────────────────────────────────────┘
```

### 各区细节

**1. 标题行**
- 左侧：`calendar` 图标（14px，主色 `#58CC02`）+ "学习日历"（14px，700，前景色）
- 右侧：本月日期范围 "MM.01 - MM.DD"（10px，`mutedForeground`），取本月1日至今天

**2. 进度环区**
- 左侧环形进度（84x84px，stroke 8px）：
  - 轨道色：`#EBEDF0`
  - 进度色：`#58CC02`（主色）
  - 填充比例 = 本月活跃天数 / 本月已过天数（含今天）
  - 圆心数字：活跃天数（26px，800，前景色）
  - 圆心副标："活跃天"（9px，`mutedForeground`）
- 右侧文案区：
  - 主标题（13px，700）："本月坚持得不错"（根据活跃比例动态切换，见下）
  - 副文案（11px，`mutedForeground`，行高 1.5）："X月已过 N 天，你练习了 M 天\n继续保持，养成学习好习惯"

**3. 近7天条形图区**
- 顶部分隔线：1px solid `border` 色
- 左侧标题："近 7 天"（12px，700）
- 右侧统计："共 X 题 · 活跃 Y 天"（10px，`mutedForeground`）
- 条形图：7 根柱子，flex 等分，gap 6px
  - 柱高 = 当日题数 / 近7天最大题数 × 100%，最低高度 8%（0题时）
  - 柱内底部显示题数（9px，700，颜色按背景深浅）
  - 颜色规则（复用现有梯度）：0题 `#EBEDF0`，1-5题 `#9BE9A8`，6-15题 `#58CC02`，>15题 `#46A302`
  - 今天：主色描边 1.5px
- 柱下方星期标签（9px）：周一至周日，今天用主色加粗

### 动态文案逻辑

根据本月活跃比例（活跃天 / 已过天数）切换主标题：

- 比例 ≥ 80%："本月坚持得不错"
- 50% ≤ 比例 < 80%："保持节奏，继续加油"
- 比例 < 50%："还有时间，今天就开始吧"

## 数据计算

所有数据从 `user.dailyHistory: DailyStat[]` 计算，无需新增接口。

- **本月活跃天数**：筛选 `date` 在本月（YYYY-MM 与今天同月）且 `questions > 0` 的记录数
- **本月已过天数**：今天的日期（1-31）
- **近7天数据**：以今天为终点倒推7天的 `DailyStat`，缺失日期按 0 题处理
- **近7天总题数 / 活跃天数**：对近7天数据求和与计数
