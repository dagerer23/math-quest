# 地图关卡界面视觉重设计

> 日期：2026-06-21
> 状态：已确认，待实现
> 落地范围：先做 web（`src/`）静态可视化原型 demo 页，验证通过后再镜像移植到 taro（`platform-taro/src/`）

## 一、背景与问题

### 1.1 现状

当前地图界面（`src/components/home/` 与 `platform-taro/src/components/home/`）采用纯左右交替蛇形布局（`getZigzagPositions`），连线为单调平行斜线；每个节点下方堆叠星徽章 + 标题块 + conic 进度环，连线又从中穿过。

### 1.2 三个核心问题

1. **路线单调**：N 关全是左右交替的平行斜线，像楼梯不像"路"，缺乏探索/旅行感。
2. **信息堆叠错位**：节点下方叠进度环 + 星徽章 + 标题块，连线穿过其中，关卡一密就互相穿插重叠。
3. **缺章节层次**：数据里已有 `unitId` 字段却没被地图利用，过一关和过一片没区别，缺成就感。

## 二、设计方向（已确认）

### 2.1 视觉风格：多邻国式（忠实还原）

经多轮方案对比（赛博霓虹 / 机甲战场 / 深空遗迹 / 等距3D岛屿 / 精致扁平 等均被否），最终确定**严格还原多邻国视觉语言**，理由：教育类最贴合"算力先锋"定位、辨识度最高、落地阻力最小。

核心视觉元素：
- **圆形按钮**（非椭圆）：白顶 + 彩色描边 + 底部投影。这是多邻国最核心的视觉符号。
- **有宽度的土路**：粗圆角线 + 圆点虚线纹理。已完成段亮绿，未完成段深灰。
- **深色背景 + 柔光晕**：深蓝灰底（`#1F2A37`→`#283443`→`#3B4A5C` 渐变），当前关顶部绿光晕。
- **无 unit 章节分段**：去掉地形色块和章节 pill，统一深色背景。每关直接显示对应知识点文字。
- **每关最多三星**：按钮侧边星行，已通关点亮（金 `#FFC700`），未通关空心（灰）。
- **去 emoji**：皇冠 / 箭头 / 锁全部改矢量图形，统一精致风格。

### 2.2 信息布局（解决堆叠重叠的核心）

**三区分离**：画布横向分中区、左区、右区，路径、按钮、信息块各居其区，互不穿插。

- **中区（路径摆幅内）**：只放路径 + 按钮。路径中线居中，摆幅 ±18px。
- **左区 / 右区**：只放信息块（星行 + 知识点标签），左右交替。路径摆幅永远到不了信息区。

**远端对齐（根治文字覆盖）**：
- 右标签：右沿永远贴画布右边，宽度随文字自适应，**向左生长**——离按钮越来越远。
- 左标签：左沿永远贴画布左边，宽度随文字自适应，**向右生长**——离按钮越来越远。
- 文字越长标签越宽，但固定端不动，只朝远离按钮方向延伸。最坏情况顶到画布边，不会越过中线碰按钮。

**标签形态统一**：所有关卡标签都用深底圆角 pill（`#2C3845`/`#26303B` 底），不再有绿色高亮 pill 这种特殊形态。**字色随状态递减**：已完成/当前关浅绿字（`#A8D870`），解锁未通灰字（`#9AA5B1`），锁定暗灰字（`#5A6678`）。状态的主区分交给按钮本身。

### 2.3 状态分级（仅靠按钮）

| 状态 | 按钮 | 星 | 标签 |
|------|------|----|----|
| 已完成 | 绿描边白底（`#3D8C00`/`#FFFFFF`）+ 底部投影 | 金星（已得颗数） | 深底浅绿字 |
| 当前关 | 发光绿（`#58CC02`）+ 脉冲圈 + 向上箭头 | 空心星 | 深底浅绿字 |
| 解锁未通 | 灰描边白底（`#9AA5B1`） | 空心星 | 深底灰字（`#9AA5B1`） |
| 锁定 | 深灰底（`#3A4654`）+ 矢量锁 | 无 | 暗灰字（`#5A6678`） |
| Boss | 紫描边白底（`#7A4FBF`）+ 矢量皇冠 | 金星全亮 | 深底浅绿字 |

### 2.4 知识点文字来源

每关的知识点标签直接用 `Level.chapter` 字段（真实数据，如"5以内加减法""10以内加减法""认识钟表图形"），非编造。

## 三、组件设计

### 3.1 数据与坐标

- 复用现有 `Level` 类型（`chapter` 字段提供知识点文字，`isBoss` 区分 Boss 关，`questionCount` 提供题数）。
- 新增坐标算法 `getWindingPositions(count, width, height)`：替代 `getZigzagPositions`，生成蜿蜒路径坐标（中线居中，摆幅 ±18px），输出节点位置 + 路径段。
- 路径用三次贝塞尔曲线（与现有 `PathsCanvas` 的绘制方式契合）。

### 3.2 节点组件 `LevelNode`

按钮：圆形（白顶 + 彩色描边 + 底部投影），按状态切换描边色与发光。Boss 用更大半径 + 矢量皇冠。锁定显示矢量锁图标。当前关叠加脉冲圈 + 向上箭头。

### 3.3 星行组件

按钮侧边 3 颗星（矢量实心/空心），已通关按 `completedLevels[levelId].stars` 点亮对应颗数。

### 3.4 标签组件 `KnowledgeLabel`

知识点文字标签，深底圆角 pill，宽度随文字自适应。左右交替放置，远端对齐到画布外缘。

### 3.5 路径组件 `WindingPath`

蜿蜒绿色土路：已完成段亮绿（`#58CC02`）+ 圆点虚线高光，未完成段深灰（`#4A5663`）+ 圆点虚线。
